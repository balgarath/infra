# Kutt URL Shortener Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Kutt URL shortener with Google Workspace auth (via oauth2-proxy) to the existing Docker Compose stack.

**Architecture:** Caddy routes `to.makenashville.org` through oauth2-proxy (for admin UI/API) or directly to Kutt (for public slug redirects). Kutt shares the existing Postgres and Redis instances with isolated database and Redis DB index.

**Tech Stack:** Docker Compose, Caddy, Kutt v3.2.3, oauth2-proxy v7.7.1, Postgres, Redis

**Spec:** `docs/superpowers/specs/2026-03-19-kutt-url-shortener-design.md`

### Task 3: Update Caddyfile

**Files:**
- Modify: `Caddyfile`

- [ ] **Step 1: Add to.makenashville.org site block**

The existing Caddyfile has only the Outline site block using `{$DOMAIN}`. Add the new Kutt site block after it. The `@protected` named matcher ensures only `/`, `/api`, and `/api/*` go through auth — all other paths (slug redirects) are public.

Final Caddyfile should be:

```caddyfile
{$DOMAIN} {
	header {
		X-Frame-Options SAMEORIGIN
		X-Content-Type-Options nosniff
		Referrer-Policy strict-origin-when-cross-origin
		-Server
	}
	reverse_proxy outline:3000
}

to.makenashville.org {
	header {
		X-Frame-Options SAMEORIGIN
		X-Content-Type-Options nosniff
		Referrer-Policy strict-origin-when-cross-origin
		-Server
	}

	handle /oauth2/* {
		reverse_proxy oauth2-proxy:4180
	}

	@protected {
		path / /api /api/*
	}
	handle @protected {
		forward_auth oauth2-proxy:4180 {
			uri /oauth2/auth
			header_up X-Forwarded-Host {host}
			copy_headers X-Auth-Request-User X-Auth-Request-Email
		}
		reverse_proxy kutt:3001
	}

	handle {
		reverse_proxy kutt:3001
	}
}
```

### Task 4: Update docker-compose.yml

**Files:**

This is the core task. Add two new services (kutt, oauth2-proxy), mount the Postgres init script, and update Caddy's dependencies.

- [ ] **Step 1: Add kutt service**

Add after the `outline` service block. Key points:
- Uses `env_file: kutt.env`
- Healthcheck hits `/api/v2/health` (confirmed in Kutt source: `server/routes/health.routes.js`)
- Depends on postgres and redis being healthy
- `start_period: 30s` gives Kutt time to run DB migrations on first boot

```yaml
  kutt:
    image: kutt/kutt:v3.2.3
    restart: unless-stopped
    env_file: kutt.env
    healthcheck:
      test: ["CMD-SHELL", "wget -qO /dev/null http://localhost:3001/api/v2/health || exit 1"]
      interval: 5s
      timeout: 5s
      retries: 10
      start_period: 30s
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
```

- [ ] **Step 2: Add oauth2-proxy service**

Add after the kutt service. For local dev, use placeholder values — production credentials come from `.env.production` via the deploy script. The `OAUTH2_PROXY_UPSTREAM=static://202` means oauth2-proxy only handles auth decisions (returns 202 on success), not proxying — Caddy does the actual proxying.

```yaml
  oauth2-proxy:
    image: quay.io/oauth2-proxy/oauth2-proxy:v7.7.1
    restart: unless-stopped
    environment:
      - OAUTH2_PROXY_PROVIDER=google
      - OAUTH2_PROXY_CLIENT_ID=${OAUTH2_PROXY_CLIENT_ID:-placeholder}
      - OAUTH2_PROXY_CLIENT_SECRET=${OAUTH2_PROXY_CLIENT_SECRET:-placeholder}
      - OAUTH2_PROXY_COOKIE_SECRET=${OAUTH2_PROXY_COOKIE_SECRET:-aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa}
      - OAUTH2_PROXY_COOKIE_SECURE=true
      - OAUTH2_PROXY_EMAIL_DOMAINS=makenashville.org
      - OAUTH2_PROXY_REDIRECT_URL=https://to.makenashville.org/oauth2/callback
      - OAUTH2_PROXY_UPSTREAM=static://202
      - OAUTH2_PROXY_HTTP_ADDRESS=0.0.0.0:4180
      - OAUTH2_PROXY_REVERSE_PROXY=true
    healthcheck:
      test: ["CMD-SHELL", "wget -qO /dev/null http://localhost:4180/ping || exit 1"]
      interval: 5s
      timeout: 5s
      retries: 5
```

Note: Google Group restriction (`OAUTH2_PROXY_GOOGLE_GROUP`, `OAUTH2_PROXY_GOOGLE_ADMIN_EMAIL`, `OAUTH2_PROXY_GOOGLE_SERVICE_ACCOUNT_JSON`) is production-only config. The group and admin email are passed as environment variables in the deploy script's docker-compose template. The service account JSON key file must be SCP'd to the server and volume-mounted into the oauth2-proxy container (see Task 6).

```bash
    sudo docker compose exec -T postgres psql -U kutt kutt -c "INSERT INTO users (email, password, verified, role) VALUES ('"'"'admin@makenashville.org'"'"', '"'"'$HASH'"'"', true, '"'"'ADMIN'"'"');"
'
```

Replace `CHANGEME` with the desired admin password. This is a one-time manual step.
